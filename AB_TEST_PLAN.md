# План A/B-теста

## Цель

Сравнить текущую модель `v1` и новую модель `v2` для прогноза дефолта по
кредитным картам.

## Разделение трафика

Трафик делится случайно 50/50:

- контрольная группа: модель `v1`;
- тестовая группа: модель `v2`.

Продолжительность теста: 2-4 недели или до накопления достаточного числа
наблюдений по фактическим дефолтам.

## Метрики

Основная метрика: F1-score для класса дефолта. Она учитывает и Precision, и
Recall.

Дополнительные метрики:

- Precision для класса дефолта, чтобы контролировать число ложных отказов;
- Recall для класса дефолта, чтобы не пропускать рискованных клиентов.

## Бизнес-метрики

1. Ожидаемые финансовые потери:

```text
loss = default_probability * credit_limit * loss_given_default
```

2. Доля одобренных заявок при фиксированном уровне риска. Метрика показывает,
сколько клиентов можно одобрить без роста ожидаемых потерь.

## Статистический анализ

Для долей ошибок и Precision/Recall можно использовать z-test для двух долей.
Для средних потерь можно использовать t-test. Доверительные интервалы строятся
на уровне 95%.

Критерий успеха: модель `v2` показывает F1-score не ниже `v1`, а бизнес-метрика
ожидаемых потерь лучше или не хуже при той же доле одобрений.
